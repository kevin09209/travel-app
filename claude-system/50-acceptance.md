## 2. 「一次糾正」的定義（寫死）

**算一次**：使用者為了把已交付或進行中的結果拉回正軌而發的訊息——指出錯誤（「不對」
「錯了」）、要求返工（「重做」「不是這樣」）、重申被忽略的要求（「我剛才說過要 X」）。
**不算**：新需求（「順便再加 Y」）、回答模型的提問、單純說「好」「繼續」。
同一則訊息糾正多個點，仍計一次。

## 4. 盲測流程（寫死，六步）

1. 選一個黃金任務，準備兩份派工 prompt：**甲版**開頭加一句「先讀 `D:\AI\CLAUDE.md`
   並遵守其鐵律與引用檔」；**乙版**不提任何制度檔。其餘文字完全相同。
2. 各派一個 fresh `general-purpose` subagent（model 同級，日常用 `sonnet`），
   在互不相干的產出路徑各跑一次。
3. 用當下時間秒數的奇偶決定呈交順序（奇＝甲在前），把順序記在第 3 節備註，
   但**寫進評分 prompt 的只有「產出一」「產出二」**。
4. 派 fresh `sonnet` subagent 當評分者。評分 prompt 只含：任務原始要求、驗收條件、
   兩份產出。優先只給這三樣，而不是給背景故事——提到「制度」「新舊版」即不盲，作廢重評。
5. 評分維度（寫死）：正確性 0–4（驗收條件逐條核對，全過＝4）；完整性 0–3（要求的都
   做了＝3）；需人工介入 0–3（以「驗收時必須追問或補救的點」計：0 點＝3 分、1 點＝2、
   2 點＝1、≥3 點＝0）。總分 10。
6. 判定：分差 ≥2 才算勝負，<2 記「平」。分數與判定寫進第 3 節表格。

**正例**：評分 prompt 只有任務要求＋驗收條件＋兩份產出。
**反例**：評分 prompt 寫「產出一有套用我們的新制度」——評分者已不盲，該次作廢。
